在一张静止的照片里，时间是被冻结的。但在 Live Photo 里，时间是在呼吸的。

苹果推出的这个特性，捕捉了快门按下前后各 1.5 秒的瞬间。正是这 3 秒钟的流动，让原本平面的记忆变得立体——或是海浪拍打礁石的微响，或是风吹过树梢的律动，亦或是爱人转头时那抹未及散去的笑意。

### 瞬时的魅力

在 Whisper 这个空间里，我一直在追求一种“静中带动”的生命感。Live Photo 完美契合了这种理念。它平时静默如迷，当你按下（或悬停）时，它便悄然苏醒，诉说那层被隐藏的故事。

### 技术实现：复刻原生体验

为了在网页上还原这种体验，我开发了一个自定义的 `<LivePhoto />` 组件。它并不是简单的视频播放器，而是模仿了 iOS 的逻辑：

1. **首帧加载**：优先加载静态 JPG 封面，确保页面性能和视觉稳定性。
2. **按需激活**：只有在用户主动交互（Hover 或 Long Press）时，才会播放关联的视频流。
3. **视觉反馈**：保留了标志性的 “LIVE” 圆环和微弱的毛玻璃质感。

### 如何使用？

如果你也想在 Whisper 中展示实况照片，可以遵循以下简单的导出流程：

#### 1. 导出原片
在 macOS 的照片应用中：选中照片，点击 **文件 > 导出 > 导出未经修改的原片 (Export Unmodified Original)**。你将获得一个 `.heic` (或 `.jpg`) 和一个 `.mov`。

#### 2. 格式转换
为了 Web 端的加载速度：将 `.heic` 转换为高压缩率的 `.jpg` 或 `.webp`，并将 `.mov` 转换为 `.mp4`（推荐使用 `ffmpeg`）。

#### 3. 放置素材
将文件放入项目的 `public/assets/` 目录下，然后在 MDX 中引用：

```tsx
import { LivePhoto } from '../../components/Common/LivePhoto';

<LivePhoto 
  client:load
  poster="/your/path/photo.jpg"
  src="/your/path/video.mp4"
/>
```
